# Premium Multi-Branch Academic Operations Platform

---

# Purpose

Dokumen ini mendefinisikan:

* arah desain UI/UX,
* prinsip visual premium,
* dashboard behavior,
* mobile-first experience,
* anti-chaos interaction,
* dan design consistency platform.

UI/UX platform ini bukan sekadar dekorasi.

UI/UX adalah:

* lapisan trust,
* lapisan kenyamanan operasional,
* dan lapisan premium experience.

---

# Core UI/UX Philosophy

---

# 1. Calm Premium Experience

Platform harus terasa:

* tenang,
* elegan,
* profesional,
* dan tidak ramai.

---

# Rule

Desain harus membantu user:

* cepat memahami situasi,
* cepat mengambil tindakan,
* dan tidak merasa kewalahan.

---

# Forbidden UI Feel

Platform tidak boleh terasa:

* murah,
* penuh warna berteriak,
* terlalu padat,
* seperti dashboard admin lama,
* atau seperti template gratisan yang ganti logo.

---

# 2. Anti-Chaos Interface

UI harus mengurangi chaos, bukan menambahnya.

---

# Rule

Setiap halaman wajib memiliki:

* fokus utama,
* hierarchy informasi,
* dan action yang jelas.

---

# Forbidden UI Behavior

Tidak boleh:

* terlalu banyak tombol utama,
* terlalu banyak popup,
* terlalu banyak tabel tanpa konteks,
* terlalu banyak alert,
* atau menu bertingkat berlebihan.

---

# 3. Mobile-First by Default

Mayoritas user penting:

* parent,
* teacher,
* student,
* bahkan admin tertentu,

akan sering memakai mobile device.

---

# Rule

Semua workflow penting wajib nyaman di mobile.

---

# Mobile Priority Workflows

* parent monitoring
* teacher schedule
* attendance
* notification
* invoice visibility
* lesson summary
* class schedule

---

# 4. Role-Based Experience

Setiap role harus terasa berbeda sesuai kebutuhan.

---

# Rule

Jangan membuat semua dashboard terlihat sama.

Dashboard harus mengikuti karakter role.

---

# Dashboard Character

| Role             | UI Character                  |
| ---------------- | ----------------------------- |
| Platform Owner   | Global SaaS Command Center    |
| Academy Director | Executive Academy Monitoring  |
| Branch Manager   | Branch Operational Control    |
| Branch Admin     | Fast Operational Workspace    |
| Teacher          | Tactical Daily Workspace      |
| Parent           | Premium Monitoring Experience |
| Student          | Simple Learning Workspace     |

* progress,

---

# Card Rules

* memiliki spacing cukup,

# Forbidden Card Behavior

# 2. Clear Information Hierarchy

Setiap halaman wajib memiliki:

```text

```

# 3. Progressive Disclosure

# Rule

# Animation Rules

# Forbidden Animation

Tidak boleh:

---

# AI Agent UI Rules

AI-generated UI wajib:

* mengikuti design direction,
* tidak membuat layout ramai,
* tidak membuat component random,
* tidak membuat style inconsistent,
* dan tidak mengorbankan mobile experience.

---

# Forbidden AI UI Behaviors

AI agents dilarang:

* membuat dashboard seragam untuk semua role,
* membuat warna sembarangan,
* membuat tabel lebar di mobile,
* membuat form panjang tanpa section,
* membuat popup berlebihan,
* atau membuat UI seperti admin template murahan.
