# Core Architectural Principles

---

# 1. Anti-Chaos First

Semua desain sistem harus membantu mengurangi chaos.

Chaos dalam konteks platform:

* jadwal bentrok,
* duplicate workflow,
* informasi tidak sinkron,
* spam komunikasi,
* kehilangan materi,
* overload admin,
* kelelahan guru,
* dan parent confusion.

---

# Rule

Jika sebuah fitur:

* menambah complexity,
* meningkatkan kebingungan,
* memperbanyak klik,
* memperbesar risiko miskomunikasi,

maka fitur tersebut harus:

* disederhanakan,
* dipindahkan,
* atau ditolak.

---

# 2. Operational Stability Over Feature Quantity

Prioritas utama platform adalah:

* stabilitas operasional,
* bukan banyak fitur.

---

# Rule

Lebih baik:

* sedikit fitur,
* tetapi workflow stabil,

daripada:

* banyak fitur,
* tetapi operasional chaos.

---

# 3. Parent Trust Is Critical

Wali murid adalah:

* payer utama,
* user sensitif,
* dan sumber trust terbesar.

---

# Rule

Semua fitur parent-facing harus:

* jelas,
* valid,
* transparan,
* dan premium.

Parent tidak boleh:

* bingung,
* merasa dibohongi,
* atau kehilangan visibilitas terhadap proses belajar anak.

---

# Required Parent Principles

Platform wajib menyediakan:

* attendance visibility,
* lesson visibility,
* schedule visibility,
* progress visibility,
* invoice transparency,
* dan notification consistency.

---

# 4. Teacher Energy Preservation

Guru adalah operational frontline.

Guru dapat:

* mengajar banyak kelas,
* lintas branch,
* menghandle ratusan murid,
* dan mengalami kelelahan operasional.

---

# Rule

Teacher workflow harus:

* cepat,
* ringan,
* minim klik,
* minim popup,
* minim repetitive task.

---

# Forbidden Teacher Experience

Platform tidak boleh:

* memperlambat guru,
* memperumit attendance,
* memperumit reporting,
* atau memperberat kelas operasional.

---

# 5. Admin Overload Prevention

Admin adalah role dengan risiko burnout tertinggi.

---

# Rule

Platform harus:

* mengurangi pekerjaan manual,
* mengurangi spam komunikasi,
* mengurangi repetitive task,
* dan membantu operational prioritization.

---

# Required Admin Principles

Admin workflow wajib:

* searchable,
* realtime,
* bulk-action friendly,
* operationally fast,
* dan tidak fragmented.

---

# 6. Multi-Branch By Default

Platform harus selalu diasumsikan:

* multi branch,
* multi operational unit,
* dan scalable.

Single branch hanyalah:

* implementasi sederhana dari multi-branch architecture.

---

# Rule

Semua:

* data,
* permission,
* scheduling,
* invoicing,
* reporting,
* dan communication

harus siap untuk:

* multi branch environment.

---

# Required Database Principle

Semua operational entities wajib mempertimbangkan:

* academy isolation,
* branch isolation,
* permission scope,
* dan branch visibility.

---

# 7. Branch Equality Principle

Tidak ada:

* branch utama,
* branch superior,
* branch pusat yang mendominasi.

Semua branch:

* setara,
* independen,
* dan memiliki otoritas operasional masing-masing.

---

# Rule

Branch:

* memiliki admin sendiri,
* invoice sendiri,
* schedule sendiri,
* operational workflow sendiri.

Academy Director hanya:

* memonitor secara global.

---

# 8. Realtime Operational Awareness

* realtime,
* sinkron,
* dan terdistribusi dengan benar.

# 10. Communication Must Reduce Noise

* jelas,
